Параллельные сортировки на CUDA

В данном репозитории представлено решение учебного задания по параллельному программированию с использованием технологии CUDA. В работе рассматривается реализация и сравнение нескольких алгоритмов сортировки при последовательном выполнении на CPU и параллельном выполнении на GPU.
Основная идея задания заключается в том, чтобы на практике изучить, как классические алгоритмы сортировки могут быть адаптированы для выполнения на графическом процессоре и какие преимущества и ограничения возникают при этом.

 Цель задания
Целью данной работы является:
освоение базовых принципов программирования на GPU с использованием CUDA;
реализация параллельных версий алгоритмов сортировки;
сравнение времени выполнения алгоритмов на CPU и GPU;
анализ эффективности параллельных вычислений при увеличении размера входных данных.

 Реализованные алгоритмы
Сортировка слиянием (Merge Sort)
Массив разбивается на подмассивы, которые обрабатываются разными блоками потоков CUDA. Каждый подмассив сортируется параллельно, после чего выполняются этапы слияния отсортированных частей. Данный алгоритм хорошо подходит для GPU, так как имеет предсказуемую структуру и эффективно масштабируется.
Быстрая сортировка (Quick Sort)
В учебной реализации массив делится на части, каждая из которых сортируется параллельно. Такой подход позволяет продемонстрировать идею параллельной быстрой сортировки на GPU, хотя полноценная рекурсивная реализация требует более сложных решений и специализированных библиотек.
Пирамидальная сортировка (Heap Sort)
Пирамидальная сортировка имеет большое количество зависимостей между элементами, что затрудняет её распараллеливание. В данной работе используется упрощённый параллельный подход, позволяющий показать особенности и ограничения данного алгоритма при выполнении на GPU.

 Тестирование и сравнение производительности
Тестирование проводилось на массивах следующих размеров:
10 000 элементов;
100 000 элементов;
1 000 000 элементов.
Для каждого размера массива измерялось время выполнения сортировки на CPU и на GPU. После выполнения сортировки дополнительно проверялась корректность результата, чтобы убедиться, что массив действительно отсортирован.
