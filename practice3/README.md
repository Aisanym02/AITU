# Описание задачи
## В данном репозитории представлено решение учебного задания по параллельному программированию на GPU с использованием технологии CUDA. В работе реализованы и сравнены несколько алгоритмов сортировки, выполненные как в последовательном режиме на CPU, так и в параллельном режиме на GPU.
Основной целью задания является практическое понимание того, как классические алгоритмы сортировки могут быть адаптированы для параллельных вычислений и как это влияет на производительность при работе с большими объёмами данных.

# Цель работы
Целью данной работы является:
реализация параллельных алгоритмов сортировки на CUDA;
сравнение времени выполнения сортировок на CPU и GPU;
анализ эффективности использования GPU при увеличении размера входных данных.
# Реализованные алгоритмы
## Сортировка слиянием (Merge Sort)
Массив разбивается на подмассивы, каждый из которых обрабатывается отдельным блоком потоков CUDA. Подмассивы сортируются параллельно, после чего выполняются этапы попарного слияния. Данный алгоритм хорошо подходит для GPU, так как имеет регулярную структуру и эффективно масштабируется.
### Быстрая сортировка (Quick Sort)
В рамках учебной реализации используется параллельный подход, при котором массив делится на части, обрабатываемые параллельно. Такой вариант позволяет продемонстрировать идею параллельной быстрой сортировки на GPU, хотя полноценная рекурсивная реализация на GPU является значительно более сложной.
Пирамидальная сортировка (Heap Sort)
Пирамидальная сортировка содержит большое количество зависимостей между элементами, что затрудняет её распараллеливание. В данной работе используется учебный вариант параллельной обработки данных, демонстрирующий ограничения данного алгоритма при переносе на GPU.
Последовательные версии
Для корректного сравнения производительности для всех алгоритмов реализованы последовательные версии, выполняемые на центральном процессоре (CPU). Это позволяет наглядно сравнить скорость выполнения CPU и GPU реализаций.
Тестирование и сравнение производительности
Тестирование проводилось на массивах следующих размеров:
10 000 элементов;
100 000 элементов;
1 000 000 элементов.
Для каждого размера массива измерялось время выполнения сортировки на CPU и на GPU. После выполнения сортировки проверялась корректность результата, чтобы убедиться, что массив отсортирован правильно.
Среда выполнения
Так как операционная система macOS не поддерживает работу с CUDA и компилятором nvcc, выполнение и тестирование программы проводилось в среде Google Colab с использованием графического процессора NVIDIA Tesla T4.
Компиляция выполнялась с помощью компилятора nvcc, запуск программы осуществлялся из среды Colab.
Выводы
По результатам выполнения задания можно сделать вывод, что сортировка слиянием показывает наилучшее ускорение при использовании GPU, особенно при больших размерах входных данных. Для быстрой и пирамидальной сортировок ускорение выражено слабее, что связано с особенностями их алгоритмов и ограничениями параллелизации.
Использование GPU оправдано при работе с большими массивами данных и алгоритмами, хорошо подходящими для параллельных вычислений.
